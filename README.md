# openshift-azure-bare-metal

IaC for deploying Azure resources for OpenShift bare-metal style nested virtualization using the `azi` CLI workflow.

## Tooling Contract

- `azi` CLI is the orchestration entrypoint.
- `azi` wraps `go-task` task libraries and Azure CLI deployment commands.
- Azure resources are configured with Azure Verified Modules (AVM) via Bicep public registry.
- Use **only** `.bicepparam` files in this repo for resource configuration.
- Do **not** add `.bicep` templates to this repo.

## `azi` CLI Quick Reference

From `azi --help` and subcommand help:

- Main usage: `azi [task] [flags]` or `azi [command]`
- Bicep workflow commands:
  - `azi bicep describe`
  - `azi bicep validate`
  - `azi bicep plan`
  - `azi bicep apply`
- AVM resource commands:
  - `azi resources avm init`
  - `azi resources avm list`
  - `azi resources avm new`

Global flags include `--dry`, `--force`, `--parallel`, `--silent`, `--summary`, `--verbose`, and `--yes`.

> `destroy` is not part of the recommended workflow in this repo because it removes deployment metadata at Azure scope, but does not guarantee deletion of underlying resources.

## How to Add New Resource Configuration

1. Initialize AVM provider data if needed:
   - `azi resources avm init`
2. List available AVM modules:
   - `azi resources avm list`
3. Generate a new resource configuration:
   - `azi resources avm new`
4. Keep generated config as `.bicepparam` under a scoped folder (`resources/`, `network/`, `compute/`, etc.).
5. Replace autogenerated required placeholders with real values.
6. Add the `.bicepparam` file path into a `deploy.yaml` deployment entry under `resources`.
7. Execute lifecycle checks:
   - `azi bicep describe`
   - `azi bicep validate`
   - `azi bicep plan`
   - `azi bicep apply`

## Manifest Model (`deploy.yaml` and `tags.yaml`)

`azi` processes manifests from your current working directory.

### Flexible repository structure

You can maintain multiple manifests in different folders and run `azi` from the directory you want to target.

Valid example layout:
- `./deploy.yaml`
- `./tenants/Contoso/subscriptions/dev/deploy.yaml`
- `./tenants/Contoso/subscriptions/test/deploy.yaml`
- `./tenants/Contoso/subscriptions/prod/deploy.yaml`
- `./tenants/Contoso/subscriptions/dev/rg-aks/deploy.yaml`
- `./tenants/Contoso/subscriptions/dev/rg-openshift/deploy.yaml`
- `./environments/dev/rg-openshift/deploy.yaml`
- `./rg-openshift/deploy.yaml`

### `deploy.yaml` (verified from `~/.azi/lib/bicep/*.yaml`)

`deploy.yaml` is parsed as a YAML list of deployment objects.

Supported keys:
- `deployment_type` (`group` or `subscription`)
- `scope`
- `desc`
- `location`
- `subscription`
- `resources` (array of `.bicepparam` paths)
- `parameters` (array of direct parameter fragments)
- `require_confirmation` (bool)
- `template_file` (unsupported in current task libs)
- `template_uri` (unsupported in current task libs)
- `template_spec` (unsupported in current task libs)

Conditional requirements:
- `deployment_type: group`
  - Required: `scope`
  - `desc` is required by validate/plan and recommended for apply naming.
- `deployment_type: subscription`
  - Required: `scope`, `location`
  - `desc` is required by validate/plan and recommended for apply naming.

Notes:
- Empty `resources` means entry is skipped by task status guards.
- `resources` and `parameters` are passed to Azure CLI as repeated `--parameters` arguments.
- Use explicit `.bicepparam` paths relative to the active manifest execution context.

### `tags.yaml` (verified from `~/.azi/lib/bicep/apply.yaml`)

`tags.yaml` is optional and used by `azi bicep apply`.

It is parsed as a YAML list of tag-operation objects with keys:
- `resource_id` (string)
- `desc` (string)
- `tags` (array of `key=value`)
- `auto_tags` (bool, default `true`)
- `action` (default `update`)
- `operation` (default `merge`)
- `require_confirmation` (bool, default `true`)

Allowed values:
- `action`: `add-value`, `remove-value`, `create`, `delete`, `update`
- `operation`: `delete`, `merge`, `replace`

Conditional requirements:
- `resource_id` required unless `action` is `create`, `add-value`, or `remove-value`.
- Must provide either non-empty `tags` or `auto_tags: true`.
- `add-value` and `remove-value` require exactly one tag.
- `delete` requires `resource_id`; `require_confirmation: false` maps to `--yes`.

For deterministic tagging behavior, always use Azure `resource_id` values.

### Example `deploy.yaml` (all available options)

```yaml
- deployment_type: group
  scope: ocp-bm-rg
  desc: Resource group baseline
  subscription: 00000000-0000-0000-0000-000000000000
  location: "" # optional for group deployments
  require_confirmation: true
  resources:
    - resources/resource-group/ocp-bm-rg.bicepparam
    - network/virtual-network/ocp-bm-vnet.bicepparam
  parameters:
    - environment=dev
    - owner=platform-team

- deployment_type: subscription
  scope: 00000000-0000-0000-0000-000000000000
  location: eastus
  desc: Subscription-scoped bootstrap
  require_confirmation: false
  resources:
    - resources/policy/policy-assignment.bicepparam
  parameters:
    - costCenter=cc1234
```

### Example `tags.yaml` (all available options)

```yaml
- desc: Merge tags into resource group and include auto Git/Azure tags
  resource_id: /subscriptions/00000000-0000-0000-0000-000000000000/resourceGroups/ocp-bm-rg
  action: update
  operation: merge
  auto_tags: true
  require_confirmation: true
  tags:
    - Environment=dev
    - Workload=openshift-bm
    - Owner=platform-team

- desc: Replace all tags on a resource
  resource_id: /subscriptions/00000000-0000-0000-0000-000000000000/resourceGroups/ocp-bm-rg/providers/Microsoft.Network/virtualNetworks/ocp-bm-vnet
  action: update
  operation: replace
  auto_tags: false
  require_confirmation: true
  tags:
    - Environment=test
    - Owner=network-team

- desc: Delete all tags from a resource without prompt
  resource_id: /subscriptions/00000000-0000-0000-0000-000000000000/resourceGroups/ocp-bm-rg/providers/Microsoft.Compute/virtualMachines/ocp-bm-host1-vm
  action: delete
  operation: delete
  auto_tags: false
  require_confirmation: false
  tags: []

- desc: Create a standalone tag name/value pair
  action: create
  auto_tags: false
  require_confirmation: true
  tags:
    - CostCenter=cc1234

- desc: Add one value to an existing tag name
  action: add-value
  auto_tags: false
  require_confirmation: true
  tags:
    - Environment=prod

- desc: Remove one value from an existing tag name
  action: remove-value
  auto_tags: false
  require_confirmation: true
  tags:
    - Environment=test
```
